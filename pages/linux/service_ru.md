# service

> Управление сервисами путём запуска скриптов инициализации.
> Предполагается, что скрипт расположен в каталоге /etc/init.d/, эту часть имени файла можно опустить.

- Запустить/остановить/перезапустить/перезагрузить сервис(скрипт должен поддерживать как минимум команды start/stop):

`service {{init_script}} {{start|stop|restart|reload}}`

- Перезапустить скрпит, выполнив его сначала с командой stop, затем с командой start:

`service {{init_script}} --full-restart`

- Вывести текущий статус сервиса:

`service {{init_script}} status`

- Вывести статусы всех сервисов:

`service --status-all`
